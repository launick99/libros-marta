# Aplicaciones para Dispositivos Móviles
## Cursada: 1° Cuatrimestre 2025

### Requisitos Obligatorios
- El objetivo del final es la creación de un sitio **realizado con Vuejs 3 y API Options**, en cuyo caso, el sistema debe estar implementado con **Vite, Eslint, Prettier y Router.**
- Pueden continuar con la **del segundo parcial o ser un tema diferente justificando la elección del tema.**
- El criterio de desarrollo de la **aplicación debe corresponder con los contenidos vistos en la cursada**, (remitirse a las clases). **Solo se evaluarán aquellos contenidos que se han dictado a este nivel de la carrera.**
- Uso de la **API LocalStorage de HTML5 para realizar un ABM** (Alta, Baja y Modificación) de datos relacionados con la temática elegida (no se aceptan formularios de contacto). Esto permitirá que el usuario pueda manipular datos
incluso sin conexión.
- La aplicación debe poseer como **mínimo 4 vínculos navegables (serán las vistas en la carpeta views) por medio de la implementación de vue-router. No pudiendo superar el máximo de 5 vistas.** Estas representarán **el contenido de cada template de los componentes dentro de router-view.**
- **Como mínimo deben desarrollar 4 componentes (en la carpeta components) con sus respectivos methods, deben ser funcionales con la interacción usuario, es decir deben poseer funciones, no solo templates) No pudiendo superar el desarrollo de un máximo de 5 componentes.**
- El diseño de la aplicación debe reflejar un estilo propio, personalizado acorde al producto. Contenido real y en español. (no contenido simulado)
- Debe poseer **imágenes** para que resulte atractiva visualmente.
- Uso de directivas de tipo **v-if / v-else**
- Uso de **v-bind** asociado a un atributo **class** para dar estilos por **operador ternario.**
- Un **informe .pdf (digital)** que explique el flujo de funcionamiento y uso de la app. (pueden destacar comportamientos, elementos empleados, etc). Con los prints de las pantallas explicando el proceso.
### Para sumar nota:
- Incorporar la interfaz **vuetify**, aprovechando los componentes y elementos visuales
para acompañar la estética y manejo con Vuejs. (Mínimo 4 componentes de
interfaz).
- Uso de **animaciones y transiciones** con Animate.css para destacar **momentos** en
el uso del sitio (cambio de vistas, repetición de elementos por v-for, etc.) -
- Relación entre **componentes padres –hijos.** Paso de información por props y uso de **emit** para comunicar desde **componentes hijos a padres. Uso de propiedades computadas, cuando sea necesario.**
- La **base de datos y los archivos PHP** deben ser trabajados desde **localhost utilizando XAMPP**. Crear la base de datos y los archivos PHP necesarios para realizar las operaciones del **ABM** (Alta, Baja y Modificación), integrando consultas **mediante fetch y gestionando CORS** para interactuar con lo desarrollado previamente a través de **LocalStorage**.
- Diseño, funcionalidad, complejidad y lógica empleados en el desarrollo, como el uso
de imágenes optimizadas serán motivos de evaluación.
